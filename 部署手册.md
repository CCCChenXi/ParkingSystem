# ParkingSystem 部署文档

## 环境要求

| 组件 | 版本要求 |
|------|---------|
| JDK | **21** |
| MySQL | 8.0+ |
| Redis | 推荐 7.x |
| RabbitMQ | 推荐 3.x |
| Maven | 3.8+ |

---

## 基础设施启动

### MySQL

创建数据库（字符集必须为 `utf8mb4`）：

```sql
CREATE DATABASE IF NOT EXISTS parking_system
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;
```

### Redis

默认端口 6379，无密码或按需配置。

### RabbitMQ

默认端口 5672，需要创建用户并授予权限：

```bash
rabbitmqctl add_user your_name your_password
rabbitmqctl set_permissions -p / your_name ".*" ".*" ".*"
```

## 配置说明

项目提供了配置模板：



### `application-example.yaml`（部署模板）

| 环境变量 | 默认值 | 必填 | 说明 |
|---------|--------|------|------|
| `MYSQL_PASSWORD` | — | **是** | MySQL 密码 |
| `RABBITMQ_USER` | — | **是** | RabbitMQ 用户名 |
| `RABBITMQ_PASSWORD` | — | **是** | RabbitMQ 密码 |
| `JWT_SECRET` | — | **是** | JWT 签名密钥 |
| `RABBITMQ_HOST` | `localhost` | 否 | RabbitMQ 主机 |
| `REDIS_PASSWORD` | 空 | 否 | Redis 密码 |

**部署前复制模板并填入环境变量：**

```bash
cp src/main/resources/application-example.yaml src/main/resources/application.yaml
# 编辑 application.yaml 或通过环境变量传入
```

## 构建与启动

```bash
# 构建
mvn clean package -DskipTests

# 启动
java -jar target/ParkingSystem-1.0.0.jar
```

## 启动缓存预热

项目启动时自动完成以下 Redis 缓存初始化（无需人工干预）：

| 缓存 | 来源 | 时机 |
|------|------|------|
| `parking:lot:info:{id}` | 全量停车场信息 | `@PostConstruct` |
| `parking:lot:list:all` | 全量停车场列表 | `@PostConstruct` |
| `parking:spot:list:{lotId}` | 全量车位静态列表 | `@PostConstruct` |
| `parking:spot:status:{lotId}` | 车位实时占用 Bitmap | `@PostConstruct` |
| `coupon:seckill:stock:{id}` | 秒杀优惠券库存 | `@PostConstruct` |

此外，系统内置 3 个定时任务自动维护缓存：

| 任务 | 频率 | 用途 |
|------|------|------|
| `DashboardPreAggregationTask` | 每 5 分钟 | 预计算今日订单数/收入，写入 Redis |
| `CacheConsistencyTask` | 每 10 分钟 | 全量重建可用车位 + 校验全量停车场列表 |
| `CouponCleanupTask` | 每天 3:00 | 清理过期优惠券的 Redis 残留 key |

## 常见问题

**启动后车位状态异常？**
项目启动时 `ParkingDataProvider.initCache()` 会自动从数据库重建所有 Redis 缓存。如果依然异常，确认 Redis 连接正常后重启即可。

**RabbitMQ 连接失败？**
确认 RabbitMQ 服务已启动，用户名密码与配置一致。如果不需要消息队列，可临时注释 `@RabbitListener` 注解调试。
